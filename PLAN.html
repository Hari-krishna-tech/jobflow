JobFlow

A personal, AI-assisted job application tracker. Add jobs manually, let AI extract structure from pasted descriptions, connect Gmail, and automatically detect job-related emails — updating status and surfacing action items in one unified dashboard.

01

Product Vision

What JobFlow is, and why it matters.

Job tracking today is fragmented: jobs live in a spreadsheet, recruiter emails pile up in Gmail, and follow-ups (OAs, interviews, offers) slip through the cracks. JobFlow unifies these into a single source of truth — combining manual tracking, AI extraction, and automatic Gmail monitoring so a candidate never misses a follow-up.

🗂️Track

Add and organize applications with full lifecycle status tracking.

🤖Extract

Paste a job description and let AI autofill company, role, salary, location.

📧Monitor

Connect Gmail; job emails are detected, classified, and matched to jobs.

⚡Act

Action items are generated from emails — never miss an interview slot.

📊Analyze

Conversion rates, response times, and company insights at a glance.

👤Personal CRM

A recruiter and company memory that grows with your search.

ℹ️
Why this is a strong portfolio projectIt combines six real integrations (Gmail, OAuth, AI, DB, scheduling, analytics) into a cohesive product that solves a genuine personal pain point — and is realistic for a solo developer to ship.

Status Lifecycle

One controlled vocabulary across jobs, emails, and AI outputs.

Applied → Assessment → Interview → HR Round → Offer
Rejected or Ghosted — terminal states reachable from any stage.
!
Constrain the AIAlways force the classifier to emit one of these exact enum values via structured JSON. Never let it free-text a status — that breaks matching and dashboard counts.

Authentication

Auth.js with Google OAuth, extended to capture Gmail tokens.

  • Google login via Auth.js (NextAuth v5).
  • Request scopes: openid, email, profile, gmail.readonly.
  • On callback, persist refresh_token & access_token to the User record.
  • Middleware guards all (dashboard) routes; redirects unauthenticated users to login.
  • Access tokens are short-lived — refresh transparently inside the Gmail service.
🔒
SecurityTokens must be stored server-side only (never sent to the client). Consider encrypting refresh_token at rest using an app-level key. Never expose tokens in client components or logs.

Gmail Integration

The MVP's highest-value feature.

Sync strategy — MVP (polling)

  • Triggered on dashboard refresh and via Vercel Cron (e.g. every 30 min).
  • Fetch the latest ~100 messages since last_sync_at.
  • Dedupe via gmail_message_id (unique constraint → idempotent).
  • Pass each to the AI classifier, match company → Job, persist.

Sync strategy — Production (push, deferred)

  • Gmail Push Notifications via Google Pub/Sub.
  • Real-time webhook on new messages — no polling.
  • Requires a public HTTPS webhook + Pub/Sub topic setup.

AI Email Classification

The engine that makes Gmail monitoring automatic.

For each fetched email, the AI returns a structured decision. Cost per email is negligible on Qwen 3 8B.

Classifier prompt

// system
You classify emails for a job-application tracker.
Analyze the email and return JSON only.

{
  "job_related": true | false,
  "company": "Amazon",
  "status": "INTERVIEW",        // enum only
  "action_required": "Schedule interview",
  "summary": "Shortlisted for SDE interview."
}

Example

📨Inbound email

"Congratulations! You have been shortlisted for the next round. Please use the link to book a slot."

🤖AI responseQwen 3 8B
{
  "job_related": true,
  "company": "Amazon",
  "status": "INTERVIEW",
  "action_required": "Schedule interview",
  "summary": "Shortlisted — book interview slot."
}
🔗
Matching logicCompany from AI is fuzzy-matched against the user's tracked Jobs (case-insensitive, alias-aware). If no match, the email is stored as job_id = null and surfaced in an "unmatched" queue for manual triage.

Implementation Order

Build it, in this order.

  1. Scaffold project
    create-next-app, Tailwind, shadcn/ui init, TypeScript strict, Prisma + Neon connection.
  2. Define Prisma schema & migrate
    User, Job, Email, Task + JobStatus / TaskSource enums. Push to Neon.
  3. Wire Auth.js + Google OAuth
    Login, session, middleware guard. Store user record on first login.
  4. Job CRUD + repository layer
    Route handlers, Zod schemas, repositories, Server Actions where sensible.
  5. Dashboard + jobs UI
    Stat cards, jobs table, status badges, dark mode.
  6. Gmail OAuth + token storage
    Add gmail.readonly scope, callback handler, encrypted token persistence.
  7. Gmail sync service
    Fetch, dedupe by message id, store raw emails. Resync button.
  8. AI classification integration
    OpenRouter client, Qwen 3 8B, structured JSON via response schema. Per-email classification.
  9. Company matching + status updates
    Fuzzy match to Jobs, update status, auto-create tasks.
  10. Job detail view + timeline
    Unified view of info, emails, tasks, status history.
  11. Tests, rate limiting, hardening
    Vitest unit tests for services/repositories, rate-limit AI + sync endpoints, error boundaries.
  12. Deploy to Vercel
    Env vars, Neon prod DB, Vercel Cron for scheduled syncs.
